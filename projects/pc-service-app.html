01Overview
Context
Developed as the final project for SEG2505 at the University of Ottawa. Applied software engineering principles end-to-end: requirements gathering, UML modeling, agile iteration, and Android implementation with a live backend.
Problem
PC repair shops manage orders across multiple staff roles with no unified system — leading to inventory mismatches, unclear order status, and manual handoffs.
Solution
A role-based Android app where each actor sees only their relevant interface. Orders flow through defined states; inventory updates in real time via Firebase Firestore.
02Role System
🛡️
Administrator
Full system access. Manages users, monitors all orders, resolves conflicts. Can override any order state.
📦
StoreKeeper
Manages inventory. Approves or rejects part requests. Updates component availability in real time.
🔧
Assembler
Receives assembly tasks. Marks progress, requests parts, and updates build status through the workflow.
🖥️
Requester
Submits custom PC build orders. Tracks order status and communicates requirements through the app.

05Code Highlights
OrderController.javaJava
public void submitOrder(Order order, String requesterId) {
    order.setStatus(OrderStatus.PENDING);
    order.setRequesterId(requesterId);
    order.setTimestamp(System.currentTimeMillis());
    db.collection("orders")
        .add(order.toMap())
        .addOnSuccessListener(ref -> { order.setId(ref.getId()); notifyStoreKeepers(order); })
        .addOnFailureListener(e -> Log.e(TAG, "Order submit failed", e));
}
RoleRouter.javaJava
public static void routeUser(Context ctx, String role) {
    Intent intent;
    switch (role) {
        case "ADMIN":       intent = new Intent(ctx, AdminActivity.class); break;
        case "STOREKEEPER": intent = new Intent(ctx, StoreKeeperActivity.class); break;
        case "ASSEMBLER":   intent = new Intent(ctx, AssemblerActivity.class); break;
        default:            intent = new Intent(ctx, RequesterActivity.class); break;
    }
    ctx.startActivity(intent);
}
InventoryManager.javaJava
public void listenToInventory(InventoryCallback callback) {
    db.collection("inventory")
        .whereEqualTo("available", true)
        .addSnapshotListener((snapshots, error) -> {
            if (error != null || snapshots == null) return;
            List<Component> items = new ArrayList<>();
            for (QueryDocumentSnapshot doc : snapshots)
                items.add(doc.toObject(Component.class));
            callback.onInventoryUpdated(items);
        });
}
