01Overview
Concept
Terry Torry is a browser-based territory game where the player and AI opponents compete to claim cells on a grid. The entire game — logic, rendering, AI, particles — lives in a single HTML file with no build step required.
Gameplay
Move your cursor to paint territory across the grid. Enemies do the same from their starting corners. The game ends after 2 minutes — highest territory count wins.
Philosophy
Built as a deliberate constraint exercise: no npm, no bundler, no library. Everything from first principles using the Canvas 2D API and raw JS.

03Game Systems
🗺️
Territory Grid
NxN cell grid with ownership state per cell. Rendered via Canvas 2D with per-cell color based on ownership.
🤖
Random Walk AI
AI uses weighted random walk: neutral cells have higher weight, already-owned territory has lower weight. Creates organic-feeling expansion.
✨
Particle System
Each cell claim spawns a burst of particles with velocity, decay, and color inherited from the claimer. Pooled to avoid GC pressure.
⏱️
Game Loop
requestAnimationFrame loop with fixed-timestep logic update and variable-rate rendering. Delta-time smoothing for consistent speed.
04Code Highlights
Particle SystemJavaScript
const POOL_SIZE = 300;
const pool = Array.from({ length: POOL_SIZE }, () => ({ active: false }));

function spawnParticles(x, y, color) {
  let spawned = 0;
  for (const p of pool) {
    if (p.active) continue;
    p.active = true; p.x = x; p.y = y;
    p.vx = (Math.random() - 0.5) * 4;
    p.vy = (Math.random() - 0.5) * 4;
    p.alpha = 1; p.color = color;
    p.decay = 0.03 + Math.random() * 0.04;
    if (++spawned >= 12) break;
  }
}
AI Random WalkJavaScript
function aiMove(ai) {
  const neighbors = getNeighbors(ai.x, ai.y);
  const weights = neighbors.map(([nx, ny]) => {
    const cell = grid[ny][nx];
    if (cell.owner === ai.id) return 0.05;
    if (cell.owner === null)  return 1.0;
    return 0.3;
  });
  const total = weights.reduce((a, b) => a + b, 0);
  let r = Math.random() * total;
  for (let i = 0; i < neighbors.length; i++) {
    r -= weights[i];
    if (r <= 0) { claimCell(...neighbors[i], ai); return; }
  }
}
Game LoopJavaScript
const TICK_MS = 100;
let accumulator = 0, lastTime = 0;

function loop(timestamp) {
  const dt = timestamp - lastTime; lastTime = timestamp;
  accumulator += dt;
  while (accumulator >= TICK_MS) {
    if (!gameOver) { aiMove(aiPlayer); checkWinCondition(); }
    accumulator -= TICK_MS;
  }
  render();
  requestAnimationFrame(loop);
}
05Design Decisions
Single file
All HTML, CSS, and JS in one file. No build step, no server needed — drag into a browser and play.
Canvas over DOM
Canvas 2D API for rendering allows full frame control and makes particle effects trivial to implement without layout reflows.
Object pool
Particles are pooled rather than allocated per-spawn. At 60fps with frequent claims, GC pauses from continuous allocation cause visible stutters.
Fixed AI tick
The AI moves on a fixed timer decoupled from the render loop. AI speed stays consistent regardless of frame rate.
06Outcomes
→Fully playable game in a single HTML file. Runs in any modern browser with no installation.
→Particle system maintains 60fps without GC stutters using object pooling — tested on Chrome, Firefox, and Safari.
→Demonstrated that game-dev concepts (game loop, AI, particles) can be implemented cleanly without any engine.
→Collaborative human–AI pair programming experiment — algorithmic refinement done in collaboration with Claude.
