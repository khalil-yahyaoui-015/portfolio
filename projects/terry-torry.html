01 Overview
Concept
Terry Torry is a browser-based territory game where the player and an AI opponent compete to claim cells on a grid. The AI uses a random-walk algorithm with territory-biased decision making. The entire game — logic, rendering, AI, particles — lives in a single HTML file with no build step required.
Gameplay
Expand from your starting position by moving into adjacent cells. The AI does the same from the opposite corner. The game ends when all cells are claimed — highest territory count wins.
Philosophy
Built as a deliberate constraint exercise: no npm, no bundler, no library. Everything implemented from first principles using the Canvas 2D API and raw JS. A proof that browser APIs alone are enough for polished interactive experiences.

03 Game Systems
🗺️
Territory Grid
NxN cell grid with ownership state per cell. Rendered via Canvas 2D with per-cell color interpolation based on ownership ratio.
🤖
Random Walk AI
AI uses weighted random walk: neutral cells have higher weight, already-owned territory has lower weight. Creates organic-feeling expansion.
✨
Particle System
Each cell claim spawns a burst of particles with velocity, decay, and color inherited from the claimer. Pooled to avoid GC pressure.
🎮
Input Handler
Keyboard (WASD/arrows) and mouse click support. Input is decoupled from the game loop via event queuing.
⏱️
Game Loop
requestAnimationFrame loop with fixed-timestep logic update and variable-rate rendering. Delta-time smoothing for consistent speed.
📊
Score & State
Live territory percentage HUD. Win/lose detection triggers a particle explosion and end-screen overlay, both drawn on canvas.
04 Code Highlights
terry-torry.html — Particle System JavaScript 
// Particle pool — reuse objects to avoid GC spikes
const POOL_SIZE = 300;
const pool = Array.from({ length: POOL_SIZE }, () => ({ active: false }));

function spawnParticles(x, y, color) {
  let spawned = 0;
  for (const p of pool) {
    if (p.active) continue;
    p.active = true;
    p.x = x; p.y = y;
    p.vx = (Math.random() - 0.5) * 4;
    p.vy = (Math.random() - 0.5) * 4;
    p.alpha = 1;
    p.decay = 0.03 + Math.random() * 0.04;
    p.color = color;
    if (++spawned >= 12) break;
  }
}

function updateParticles() {
  for (const p of pool) {
    if (!p.active) continue;
    p.x += p.vx; p.y += p.vy;
    p.alpha -= p.decay;
    if (p.alpha <= 0) p.active = false;
  }
}
terry-torry.html — AI Random Walk JavaScript 
// Weighted random walk: prefer unclaimed cells
function aiMove(ai) {
  const neighbors = getNeighbors(ai.x, ai.y);
  const weights = neighbors.map(([nx, ny]) => {
    const cell = grid[ny][nx];
    if (cell.owner === ai.id) return 0.05;   // already owned
    if (cell.owner === null)  return 1.0;    // unclaimed — preferred
    return 0.3;                              // opponent — risky
  });

  const total = weights.reduce((a, b) => a + b, 0);
  let r = Math.random() * total;
  for (let i = 0; i < neighbors.length; i++) {
    r -= weights[i];
    if (r <= 0) {
      const [nx, ny] = neighbors[i];
      claimCell(nx, ny, ai.id);
      ai.x = nx; ai.y = ny;
      return;
    }
  }
}
terry-torry.html — Game Loop JavaScript 
// Fixed timestep with delta-time accumulator
const TICK_MS = 100; // AI moves every 100ms
let accumulator = 0;
let lastTime = 0;

function loop(timestamp) {
  const dt = timestamp - lastTime;
  lastTime = timestamp;
  accumulator += dt;

  while (accumulator >= TICK_MS) {
    if (!gameOver) {
      aiMove(aiPlayer);
      checkWinCondition();
    }
    accumulator -= TICK_MS;
  }

  render();
  requestAnimationFrame(loop);
}

05 Design Decisions
Single file
All HTML, CSS, and JS in one file. No build step, no server needed — drag the file into a browser and play. Portability was a hard constraint.
Canvas over DOM
Using the Canvas 2D API for rendering instead of DOM manipulation allows full frame control and makes particle effects trivial to implement without layout reflows.
Object pool
Particles are pooled rather than allocated per-spawn. At 60fps with frequent claims, GC pauses from continuous allocation would cause visible stutters.
Fixed AI tick
The AI moves on a fixed 100ms timer decoupled from the render loop. This keeps AI speed consistent regardless of frame rate and makes difficulty tuning trivial.
06 Outcomes
→ Fully playable game in a single <100KB HTML file. Runs in any modern browser with no installation or network dependency.
→ Particle system maintains 60fps without GC stutters using object pooling — tested on Chrome, Firefox, and Safari.
→ Demonstrated that game-dev concepts (game loop, AI, particles, collision) can be implemented cleanly without any game engine or framework.
→ Collaborative development process with AI assistance for algorithmic refinement — a personal experiment in human–AI pair programming for creative projects.
